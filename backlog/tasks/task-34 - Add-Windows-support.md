## Description

<!-- SECTION:DESCRIPTION:BEGIN -->
The project currently has zero Windows support. The installer (`setup.sh`) and auto-commit hook (`backlog-commit-hook.sh`) are bash-only, and the Node.js runtime has path separator issues in exclusion matching.

**Hard blockers (bash scripts):**
- `setup.sh` is a bash script ŌĆö cannot run natively on Windows
- `backlog-commit-hook.sh` is bash ŌĆö auto-commit won't work
- `~/.local/share/backlog-setup` shared dir path ŌĆö non-standard on Windows (should be `%LOCALAPPDATA%\backlog-setup`)
- Unix commands throughout: `mktemp -d`, `chmod +x`, `cp -a`, `sed`, `read -rp`, `BASH_SOURCE[0]`

**Node.js runtime issues:**
- `exclusion.mjs` regex uses only `/` separator (`[^/]*`) ŌĆö `path.relative()` returns `\` on Windows, so exclusion matching silently fails
- `rag-server.mjs` calls `execFile()` on a `.sh` script ŌĆö auto-commit fails without bash
- One existing Windows-aware fix: `backlog-proxy.mjs:222` has `replace(/\\/g, "/")` for the `isMain` check

**What already works cross-platform:**
- Node.js MCP servers (if launched manually with correct env vars)
- LanceDB vector store and embedding model
- MCP protocol (stdio-based)
- `node:path` functions (`join`, `resolve`, `relative`)
- `os.homedir()`, `fs.watch({ recursive: true })`

### New files
- **`setup.mjs`** ŌĆö 1084-line cross-platform Node.js installer, full port of `setup.sh`. Supports all flags (`--local-cache`, `--submodule`, `--backlog-remote`, `--update`, `--yes`). MCP config paths use forward slashes via `toForwardSlash()`.
- **`lib/platform.mjs`** ŌĆö Platform helpers: `getSharedDir()` (`%LOCALAPPDATA%\backlog-setup` on Windows, `~/.local/share/backlog-setup` on Unix), `getSharedCacheDir()`, `getTempDir()`, `toForwardSlash()`, `IS_WINDOWS`.
- **`lib/backlog-commit-hook.mjs`** ŌĆö Node.js port of `backlog-commit-hook.sh`. Exports `commitBacklogChanges(backlogDir, operation)`. Detects submodule vs plain repo vs no-git. Also runnable as CLI.
